# 3090 训练推进计划：军事三维目标与二维假目标判别

日期：2026-07-06

## 1. 本机与 3090 的分工

当前本机是 RTX 3050 Ti Laptop GPU，显存约 4GB，适合做：

1. 数据集检查、物理诊断、绘图和文档更新。
2. 1-3 个随机种子的短/中量训练。
3. smoke test、单门消融、曝光匹配控制的快速复核。
4. 论文草稿和 PPT 中间版本更新。

实验室 RTX 3090 24GB 适合做：

1. 5 个随机种子以上的完整训练矩阵。
2. 60-80 epoch 的主实验和融合方式对比。
3. 更大 batch、更完整鲁棒性曲线。
4. 最终写入论文主表格的定稿结果。

因此采用如下策略：

```text
本机先推进、先发现问题、先生成初步证据；
3090 只承担本机明显耗时或需要最终定稿稳定性的实验。
```

本机脚本：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_local_paper_experiments.ps1
```

本机汇总：

```text
experiments\localgpu_combined_results.csv
writing\localgpu_training_report_2026-07-06.md
```

## 2. 为什么需要 3090

当前本机已经完成了小规模验证：44 个精选军事模型、三类军事目标识别、真实 3D 与二维平面假目标判别、hard projection、rectangular-overlap、曝光匹配和单 gate 消融。

下一步不应只追求“再跑一个更高准确率”，而应补齐论文需要的稳定性证据：

1. 多随机种子：从 3 个 seed 扩展到 5 个 seed。
2. 更长训练：主实验从 20 epoch 提高到 80 epoch，减少欠训练质疑。
3. 完整消融：full gate stack、单 gate、融合方式、迁移学习、鲁棒性同时覆盖。
4. 结果可追溯：每组实验保存 `summary.json`、`results.csv`、`aggregate_results.csv`、训练曲线和混淆矩阵。

3090 的 24GB 显存对当前网络不是为了“勉强跑得动”，而是为了把所有验证矩阵稳定、快速、成体系地跑完。

先 dry run：

```powershell

```

确认路径无误后运行全套：

```powershell

```

如果实验室时间有限，优先跑：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_3090_paper_experiments.ps1 -Stages core,ablation
```

如果要复现本机发现的前景直方图匹配负结果，单独跑：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_3090_paper_experiments.ps1 -Stages hist -AblationEpochs 40
```

如果要复现像素级几何后处理伪迹检查，单独跑：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_3090_paper_experiments.ps1 -Stages geom -AblationEpochs 40
```

这两组最直接支撑论文主张：

```text
激光选通 gate stack 不是普通单帧图像分类，而是提供跨 gate 的深度响应序列差异；
真实三维目标表现为 gate 间结构变化，平面假目标表现为同一投影轮廓的强度缩放。
```

## 6. 预期论文解释

如果 3090 结果与当前 3 seed 结果一致，论文中可以这样组织：

1. 首先展示 Blender 选通仿真和 false target 建模。
2. 然后展示 gate stack 物理诊断：flat false 跨 gate 高相关，true3d 跨 gate 结构变化明显。
3. 再用训练结果证明：hard projection 与 rectangular-overlap 中，full stack 明显优于 single gate。
4. 接着用曝光匹配和鲁棒性结果说明：模型不是简单依赖亮度，而是在利用跨 gate 响应模式。
5. 最后用三类军事目标识别和迁移学习说明：该框架可以从可控仿真迁移到小样本军事目标。

## 7. 需要重点观察的结果

| 观察点 | 理想结果 | 论文含义 |
|---|---|---|
| `rect_matched_full` | 5 seed 下均值仍明显高于单 gate | gate stack 是核心信息来源 |
| `single_gate1` | 可能高于 gate0/gate2，但应低于 full stack | gate1 有残余局部线索，但不足以替代序列 |
| `foreground/p99 matched gate1` | 不应恢复到 full stack 水平 | 排除单纯亮度或高分位亮度捷径 |
| `hist matched single gate` | 可能仍然偏高 | 如果偏高，说明单帧捷径主要来自面积、边缘和形态，而不是简单灰度直方图 |
| `geom single gate` | 如果仍然很高，不作为正结果 | 说明 PNG 后处理会产生伪迹，后续应转向 Blender 物理渲染平面假目标 |
| `random_gate_dropout` | 低于 clean full stack | 多 gate 完整性有贡献 |
| `military3class transfer` | std 低于 scratch 或均值略高 | 小样本迁移提高稳定性 |
